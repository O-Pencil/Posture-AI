# Posture-AI 技术实现文档

> 版本：v2026-06-07
> 来源：基于 PRD v2.0 与当前仓库代码复核
> 结论：以完整 React Native Android App 承载端侧 Qwen + MNN 本地 CPU 推理，云端只做辅助能力

## 1. PRD 技术评审结论

PRD 的核心产品诉求不变：面向久坐用户做“不驼背坐姿助手”，形成检测、提醒、训练、复盘闭环。需要重新裁决的是技术实现口径：

| 事项 | 最新裁决 |
| --- | --- |
| 交付形态 | 必须是完整手机 App。当前仓库为 React Native 0.76 + Android 原生混合 App，满足方向；小程序、H5、纯网页不进入初赛方案。 |
| UI 框架 | 前端 UI 明确使用 RN。Compose 只作为原生调试/面板能力，不替代主 UI。 |
| 端侧 AI | Qwen + MNN 是初赛技术主路径，核心姿态分类、分数、提醒文案必须可离线运行。 |
| Arm SME2 | MNN 运行库必须采用支持 Arm SME2 的 arm64 构建；App 需在演示/状态页展示后端能力、推理耗时和模型状态。 |
| 云端 AI | 云端 Qwen-VL/API 仅用于低频视觉评估、报告润色、失败兜底或复赛能力，不能承担实时姿态主判定。 |
| 回退策略 | MNN 模型缺失、加载失败、超时或 OOM 时回退到规则化阈值/启发式结果，但演示话术必须区分“比赛主路径”和“兜底路径”。 |

## 2. 总体架构

```text
React Native UI (App.tsx)
  |
  | NativeEventEmitter / NativeModules
  v
Kotlin Android 层
  - rn/KinematicsModule.kt
  - inference/mnn/KinematicsHub.kt
  - bluetooth/SpineBluetoothManager.kt
  - service/ServiceRuntime.kt
  - mcp/McpHttpServer.kt
  |
  | JNI
  v
C++ / MNN 层
  - eyes_mnn_bridge.cpp
  - eyes_llm_session.cpp
  - libMNN.so
  |
  v
Qwen MNN 模型本地 CPU 推理
```

核心数据流：

```text
BLE/Mock 四元数
  -> Kotlin/C++ 角度解算
  -> KinematicsHub 状态机
  -> MNN/Qwen 姿态分类与本地反馈
  -> RN 仪表盘 + 震动反馈 + MCP 工具输出
```

MCP 是对外智能体接口，不是 App 完整性的替代。比赛展示应先展示手机 App 的本地闭环，再展示 MCP 作为加分项。

## 3. 端侧 Qwen + MNN 实现

### 3.1 当前代码

| 文件 | 职责 |
| --- | --- |
| `android/app/src/main/java/com/postureai/inference/mnn/MnnPerceptionEngine.kt` | Kotlin 到 JNI 的 MNN 推理桥，封装图片、音频、Prompt 和指标读取 |
| `android/app/src/main/java/com/postureai/inference/mnn/InferenceExecutor.kt` | 单线程 `eyes-mnn-infer` 串行化模型加载和推理 |
| `android/app/src/main/cpp/eyes_mnn_bridge.cpp` | JNI 入口，处理 `nativeInit`、`runInference`、指标和角度 JNI |
| `android/app/src/main/cpp/eyes_llm_session.cpp` | MNN Transformer LLM 会话封装，设置 4 线程、低精度、低内存、token 指标 |
| `android/app/src/main/jniLibs/arm64-v8a/libMNN.so` | APK 内置 MNN 运行库 |

当前模型目录沿用历史命名：

```text
filesDir/mnn_models/qwen3-vl-2b/
```

初赛文档目标是 Qwen/Qwen3.5 2B INT4 姿态分类模型。短期可继续复用该目录名以减少代码扰动，但 README 和演示话术必须说明“目录名是历史名，技术主路径是 Qwen + MNN 端侧推理”。

### 3.2 SME2 落实要求

SME2 不是 RN 或 Kotlin 层开关，而是 MNN arm64 运行库的构建与运行能力。初赛必须补齐三件事：

1. 使用支持 Arm SME2 的 MNN 版本重新构建或替换 `libMNN.so`。
2. 保留 NEON 回退，避免非 SME2 设备无法启动。
3. 在 App 状态页或 MCP `phone_status` 输出中暴露：
   - MNN native library loaded
   - model loaded
   - backend capability: `SME2 | NEON | unknown`
   - `ttft_ms`、`prefill_ms`、`decode_ms`、`tokens_generated`、`decode_tps`

演示要求：关网状态下触发一次姿态异常，App 展示本地分类和文案，同时展示 MNN 推理指标。

### 3.3 输入输出契约

端侧模型输入应收敛为短上下文结构化 Prompt，而不是长篇自由问答：

```json
{
  "schema_v": 1,
  "window_ms": 5000,
  "neck_pitch_deg": 22.4,
  "thor_pitch_deg": 16.8,
  "lumbar_roll_deg": 3.1,
  "duration_min": 5,
  "last_state": "NORMAL"
}
```

端侧模型输出只允许结构化字段：

```json
{
  "schema_v": 1,
  "classification": "tech_neck",
  "confidence": 0.86,
  "action_id": "neck_retraction",
  "severity_level": 2
}
```

自然语言反馈由 App 本地查表生成，再经过禁词检查。这样可以满足安全审核和离线可控性。

## 4. RN App 实现边界

RN 是主 UI 层，负责：

- 首次进入、连接、校准、仪表盘、训练、日报和 F7 Mock Console。
- 订阅 `KinematicsModule` 的 `onKinematicsUpdate`。
- 展示模型状态、离线状态和推理指标。
- 调用原生模块启动 MCP Service、切换 Mock、读取最新姿态状态。

RN 不负责：

- 高频 BLE 采样。
- Qwen/MNN 推理。
- SME2 能力判断。
- 震动执行。
- 长生命周期 HTTP Server。

这些能力留在 Kotlin/C++ 层，确保 App 不是网页壳。

### 4.1 UI 视觉约定

主 App 采用克制的深色健康设备风格，不做营销页和 H5 壳体验。

| Token | 用途 | 建议值 |
| --- | --- | --- |
| `status-healthy` | 正常姿态、推理成功 | `#10B981` |
| `status-warning` | 头前倾、模型加载中、电量偏低 | `#F59E0B` |
| `status-alert` | 驼背、侧倾、API 失败 | `#EF4444` |
| `status-offline` | 传感器断连、MCP 停止 | `#6B7280` |
| `bg-canvas` | 页面底色 | `#0A0F1A` |
| `bg-surface-1` | 卡片/面板 | `#111827` |
| `text-primary` | 主文字 | `#F1F5F9` |

状态色不混用：`NORMAL` 永远是绿色，`TECH_NECK` 永远是琥珀，`SLUMPED/LEFT_LEAN` 永远是红色，`OFFLINE` 永远是灰色。不要使用紫蓝渐变、玻璃拟物或大面积营销式 hero。

### 4.2 原型与交互来源

HTML 原型保留在 `prototype/`，用于视觉和演示参考；交互权威以 PRD 的用户故事、功能需求和 §11 页面清单为准。

核心初赛页面：

| 页面 | 原型文件 | 用途 |
| --- | --- | --- |
| 书桌 / 仪表盘 | `prototype/desk.html`、`prototype/dashboard.html` | 日常监测、分数、脊柱状态 |
| 植物 / 游戏化 | `prototype/plant.html`、`prototype/gamification.html` | 初赛 P0 正反馈 |
| Agent / MCP | `prototype/agent.html`、`prototype/demo.html` | 智能体工具调用和演示 |

## 5. 云端能力边界

允许云端：

- 低频侧面照视觉评估。
- 日报/周报文案润色。
- 复赛阶段多模态能力扩展。
- 模型缺失时的演示兜底，但必须标注 `degraded_mode`。

禁止云端：

- 替代实时姿态分类主路径。
- 断网时导致主仪表盘不可用。
- 输出未经校验的医疗诊断、疗效承诺或处方建议。

## 7. 数据与安全契约

### 7.1 跨模块数据规则

所有跨模块 JSON 必须带 `schema_v`，初赛统一为 `schema_v: 1`。App 端在进入 UI 或报告前做运行时校验；固件侧只做最小字段长度、枚举和数值范围校验。

核心对象：

| 对象 | 生产者 -> 消费者 | 必填字段 |
| --- | --- | --- |
| 姿态采样 | 固件/BLE -> App | `schema_v`、`node_id`、`ts_ms`、`quat`、`battery_mv`、`seq` |
| 设备状态 | 固件 -> App | `schema_v`、`connected`、`battery_pct`、`firmware_v`、`sample_hz` |
| 震动命令 | App -> 固件 | `schema_v`、`pattern_id`、`strength`、`duration_ms` |
| 校准会话 | App 内部 | `schema_v`、`started_at`、`imu_baseline`、`duration_sec` |
| 姿态事件 | 算法/AI -> 日报 | `schema_v`、`event_type`、`started_at`、`duration_sec`、`severity` |
| AI 输出 | 云端/端侧 -> 校验器 -> UI | `schema_v`、`module`、`summary`、`items`、`confidence` |

姿态评分口径：

```text
score = round(0.5 * good_pct * 100
            + 0.3 * (1 - abnormal_minutes / total_minutes) * 100
            + 0.2 * training_completion_pct * 100)
```

### 7.2 安全链

AI 输出分云端路径和端侧路径，但最终都必须经过校验后才能展示。

云端路径用于视觉评估、日报/周报润色：

1. 输入过滤：疼痛、麻木、头晕、术后、急性、诊断、药物、手术等高风险词进入就医提示分支。
2. Prompt 约束：禁止疾病诊断、药物建议、疗效保证、医疗器械购买建议。
3. 输出校验：`summary`、`recommendations/items` 非空；置信度在 `[0, 1]`；文案不含禁词。
4. 失败兜底：不展示 AI 原文，使用本地中性建议并记录 `AI_OUTPUT_INVALID`。

端侧路径用于 Qwen + MNN 本地推理：

1. 模型只输出结构化字段：`classification`、`confidence`、`action_id`、`severity_level`。
2. App 用本地查表把 `action_id + severity_level` 翻译成自然语言。
3. 查表文案再走禁词检查，命中则替换成预置兜底文案。
4. MNN 崩溃、超时或 OOM 时回退规则化阈值，不阻塞主流程。

禁词示例：

| 类别 | 示例 |
| --- | --- |
| 诊断 | 确诊、诊断为、患有、综合征（作为结论） |
| 治疗 | 治疗、治愈、药物、手术、注射、贴片 |
| 承诺 | 保证、一定、100%、彻底、永远 |
| 营销 | 限时、优惠、推荐购买、扫码 |

## 8. 构建、运行与模型安装

### 8.1 环境要求

- Node >= 18。
- Android Studio Ladybug 或更新版本。
- Android SDK 35。
- Android NDK 26，当前项目使用 `arm64-v8a`。
- 物理 Android 设备 API 29+，建议 8GB 内存。
- 端侧模型演示设备建议使用支持 Arm SME2 的新款 Arm CPU；非 SME2 设备必须能回退到 NEON。

### 8.2 启动 App

```bash
npm install
npm start
```

另开终端：

```bash
npm run android
```

Android 原生构建：

```bash
cd android
./gradlew assembleDebug
```

### 8.3 首次运行

1. 授予摄像头、麦克风、通知、蓝牙相关权限。
2. 连接硬件或打开 F7 Mock Console。
3. 完成 5 秒校准。
4. 进入 RN 仪表盘，观察分数、姿态状态、角度和模型状态。
5. 如需 MCP，启动服务后记录 `http://<手机局域网 IP>:8765/mcp` 和 Bearer Token。

### 8.4 MNN 模型目录

当前代码默认扫描历史目录名：

```text
/data/data/com.postureai/files/mnn_models/qwen3-vl-2b/
```

推送命令：

```bash
adb push <本地模型目录>/ \
  /data/data/com.postureai/files/mnn_models/qwen3-vl-2b/
```

目录至少包含：

```text
config.json
llm.mnn
llm.mnn.weight
visual.mnn
visual.mnn.weight
```

初赛技术口径仍是 Qwen/Qwen3.5 2B INT4 姿态分类模型；`qwen3-vl-2b` 只是当前代码沿用的历史目录名。

### 8.5 常见问题

| 问题 | 处理 |
| --- | --- |
| 局域网访问不到 MCP | 检查手机和 PC 是否同 Wi-Fi、路由器是否开启 AP 隔离、端口是否为 8765 |
| `Native libraries failed to load` | 检查 APK 内是否有 `lib/arm64-v8a/libMNN.so` 和 `libeyes_mnn_bridge.so` |
| 编译找不到 `llm/llm.hpp` | 检查 `android/app/src/main/cpp/CMakeLists.txt` 的 `MNN_SOURCE_ROOT` 是否指向真实 MNN 源码 |
| Android 构建报 NDK `source.properties` 缺失 | 本机 NDK 安装不完整，重新安装 NDK 26 或修正 `local.properties` |

## 9. 初赛验收清单

- [ ] 真机可安装 APK，非小程序/非 H5。
- [ ] RN 仪表盘可离线显示姿态分数、角度、状态和本地反馈。
- [ ] F7 Mock 可在 30 秒内切换正常、驼背、头前倾、侧倾、离线。
- [ ] MNN 权重存在时可本地加载 Qwen 模型并返回结构化分类。
- [ ] MNN 权重缺失时展示 degraded mode，主流程不崩溃。
- [ ] 端侧推理指标可见：首 token、总耗时、token 数、TPS。
- [ ] SME2/NEON 后端能力可见，SME2 设备上使用支持 SME2 的 MNN 库。
- [ ] 断网演示仍能跑通姿态分类、提醒和训练入口。
- [ ] 云端 API 失败不会影响实时姿态主链路。
