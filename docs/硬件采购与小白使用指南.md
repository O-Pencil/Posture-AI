# Catune 硬件采购与小白使用指南

> 版本：v2026-06-07
> 目标读者：没有硬件基础，但需要在 2 周初赛内跑通“手机 App + 1 个 IMU 姿态节点 + 震动反馈”的人
> 原则：先 USB 供电、先 1 节点、先免焊/少焊；跑通后再做电池、外壳和 3 节点

## 1. 先买什么

### 1.1 最小可演示套装

这套用于先跑通 Node-T 胸椎节点：ESP32-S3 读 BNO085 姿态数据，通过 BLE 发给手机 App。

| 优先级 | 名称 | 数量 | 建议规格 / 搜索关键词 | 用途 | 备注 |
| --- | --- | ---: | --- | --- | --- |
| 必买 | ESP32-S3 开发板 | 1 | `ESP32-S3-DevKitC-1 N16R8` 或 `ESP32-S3 N16R8 DevKit` | 主控，负责 BLE、读取 IMU、控制震动 | 认准 S3；优先 N16R8 或 N8R8，带 PSRAM 更稳 |
| 必买 | BNO085 / BNO086 IMU 模块 | 1 | `BNO085 STEMMA QT Qwiic`、`BNO086 I2C 模块` | 姿态传感器，输出四元数/欧拉角 | BNO085 自带姿态融合，比裸 MPU6050 省很多算法时间 |

## 2. 推荐购买组合

### 方案 A：最快跑通，适合零基础

只买：

1. ESP32-S3-DevKitC-1 N16R8 x1
2. BNO085/BNO086 I2C 模块 x1
3. 杜邦线 + 面包板 + USB-C 数据线
4. 魔术贴 + 3M 泡棉胶

优点：便宜、最少坑、当天能接线测试。  
缺点：暂时没有真实震动，线比较散。

### 方案 B：初赛演示推荐

在方案 A 基础上加：

1. DRV2605L 震动驱动 x1
2. 3V 扁平震动马达 x1
3. 小项目盒 x1

优点：能演示“异常姿态 -> 手机提醒 -> 身体震动”的闭环。  
缺点：马达线可能需要夹线或简单焊接。

### 方案 C：复赛/成品化

在方案 B 基础上加：

1. BNO085/BNO086 再买 2 个，做颈椎/腰椎节点
2. LiPo 电池 + TP4056 充电保护板 + 开关
3. 更稳的绑带和外壳

优点：更像完整产品。  
缺点：调试时间明显增加，不建议初赛第一版直接做。

## 3. 这些硬件分别是什么

### ESP32-S3 开发板

可以理解为“小电脑”。它负责：

- 通过 I2C 读取 BNO085 姿态数据。
- 通过 BLE 蓝牙把姿态发给 Android App。
- 接收 App 命令后控制震动。
- 初赛阶段也可以生成 Mock 姿态数据。

### BNO085 / BNO086

可以理解为“会自己算姿态的传感器”。普通 MPU6050 只给原始加速度/陀螺仪，新手还要自己写融合算法；BNO085 可以直接输出四元数/旋转向量，更适合两周比赛。

### DRV2605L + 震动马达

DRV2605L 是“震动马达控制器”。不要直接把马达接 ESP32 的 GPIO，马达启动电流会让开发板不稳定。用 DRV2605L 后，ESP32 只发 I2C 命令，马达由驱动芯片控制。

## 4. 第一次接线

### 4.1 ESP32-S3 接 BNO085

先只接 4 根线：

| BNO085 引脚 | 接到 ESP32-S3 | 含义 |
| --- | --- | --- |
| VIN / 3V | 3V3 | 供电 |
| GND | GND | 地线 |
| SDA | GPIO8 | I2C 数据 |
| SCL | GPIO9 | I2C 时钟 |

注意：

- 只用 3.3V，不要接 5V。
- GND 必须接，否则数据会乱。
- 如果你的开发板 GPIO8/9 被占用，可以换其他 GPIO，但固件里也要同步改。

### 4.2 加 DRV2605L 震动

DRV2605L 也是 I2C，可以和 BNO085 共用 SDA/SCL：

| DRV2605L 引脚 | 接到 ESP32-S3 | 含义 |
| --- | --- | --- |
| VIN | 3V3 | 供电 |
| GND | GND | 地线 |
| SDA | GPIO8 | I2C 数据 |
| SCL | GPIO9 | I2C 时钟 |
| Motor + / - | 震动马达两根线 | 马达输出 |

一个 I2C 总线上可以挂多个模块，只要地址不冲突。BNO085 和 DRV2605L 常见地址不同，一般可以共用。

## 5. 第一次通电

1. 先不要接电池，只用 USB-C 数据线连接电脑。
2. 开发板亮灯后，先在 Arduino IDE / ESP-IDF 里跑一个 blink 示例。
3. 再跑 I2C scanner，确认能扫到 BNO085。
4. 再跑 BNO085 示例，确认串口能看到四元数或欧拉角。
5. 最后再打开 Android App，看 BLE 姿态是否进入 RN 仪表盘。

如果第 3 步扫不到：

- 检查 SDA/SCL 有没有接反。
- 检查 VIN 是否接 3V3，GND 是否接 GND。
- 换一根 USB 数据线。
- 降低 I2C 频率到 100kHz。

## 6. 佩戴方式

初赛只需要 1 个 Node-T 胸椎节点：

1. 把 BNO085 固定在上背部中线，大概肩胛骨之间偏下的位置。
2. 模块箭头/文字朝外，尽量不要歪。
3. 用魔术贴绑带或泡棉胶固定，避免走动时晃。
4. 打开 App 后保持自然坐姿 5 秒，作为零位校准。
5. 演示驼背时，身体向前塌 10-20 度，App 应该在几秒内切到异常状态。

## 7. 新手最容易踩的坑

| 问题 | 表现 | 处理 |
| --- | --- | --- |
| USB 线只能充电 | 电脑没有串口 | 换明确支持数据传输的 USB-C 线 |
| SDA/SCL 接反 | 扫不到 I2C 地址 | 对调 SDA/SCL |
| 没接 GND | 数据乱跳 | 所有模块 GND 必须共地 |
| 用 5V 接传感器 | 模块发热或损坏 | 全部按 3.3V 接 |
| 传感器绑得太松 | 姿态数据抖动 | 用泡棉胶 + 魔术贴固定 |
| 直接 GPIO 带马达 | 开发板重启 | 用 DRV2605L 或三极管/MOSFET 驱动 |
| 先做电池 | 充电/短路风险 | 初赛第一版先 USB 供电 |
