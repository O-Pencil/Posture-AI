## 1. 为什么当前代码暂不集成端侧模型

初赛第一优先级是「RN 框架能在模拟器跑起来 + 有基础页面显示关键数据」。端侧 Qwen + MNN 推理在当前开发机上**无法落地**，原因有二：

1. **模型装不下**：Qwen 2B INT4 权重（约 1.2–1.5GB）+ MNN 运行库不适合放进当前开发/CI 环境，也无法推到模拟器跑通。
2. **原生构建缺源**：原 `CMakeLists.txt` 把 `MNN_SOURCE_ROOT` 硬编码到仓库外的 `Template-github/eyes-on-my-phone/MNN-master`，该路径在本仓库不存在，导致 `externalNativeBuild` 必然失败，`assembleDebug` 连 APK 都产不出来。

因此初赛阶段**先把端侧推理从主链路移除**，让 App 成为「纯 RN + 轻量 Kotlin」可构建、可在 x86_64 模拟器运行的形态，由本计划规划重新接入。

**未恢复（确属 Non-Goals，留在 git `d17af44`）**：`DefaultPerceptionEngine`（相机/音频编排）、`HeuristicAnalyzer`、`PerceptionEngine` 接口、`ModelInstallInfo`（且原文件有损坏字符串）、MCP/Watchdog/Camera/Audio。

**仍缺（必须在有算力的机器/真机上补）**：

1. **MNN 源码** → 放到 `android/app/src/main/cpp/third_party/MNN/`（含 `include/MNN`、`llm/llm.hpp` 等），否则 `-PenableMnn` 构建报 FATAL_ERROR。
2. **SME2 版 libMNN.so**（替换现有 arm64 库）。
3. **Qwen 2B INT4 模型**（`config.json` / `llm.mnn` / `llm.mnn.weight`），push 到设备。
4. **接线 + 姿态化**：Settings 调试卡片已调用 `CatuneMnn.getStatus()` / `inferText()`；真机模型跑通后，再把模型文案接入 TS 姿态建议流。

> 注：当前姿态主链路已经迁到 TS：`src/posture/engine.ts` 负责规则分类、打分、建议兜底；`src/posture/sensorSource.ts` 负责手机 IMU 数据源。端侧模型只补建议文案，不负责安全底线。

## 3. 目标架构（重新接入后）

```text
RN UI (AppShell / Settings)
  ├─ Desk/Plant：TS 规则状态机（src/posture/engine.ts）
  │    ▲
  │    └─ sensorSource.ts(iPhone/Android DeviceMotion) 或 mock.ts
  └─ Settings MNN DEBUG
       └─ NativeModules.CatuneMnn.getStatus() / inferText(prompt)
            └─ MnnDebugModule → MnnPerceptionEngine → InferenceExecutor
                 └─ JNI libposture_ai_bridge.so → libMNN.so → Qwen MNN 模型
```

**纪律（沿用 PRD §5.10）**：姿态分类/提醒走 TS 规则引擎，模型只做「文案生成」这类规则做不到的事；模型超时/崩溃/OOM 自动回退规则引擎；模型输出必须过安全链。

## 3.1 iPhone 跑端侧小模型评估

iPhone 当前适合继续承担真实手机 IMU 数据测试，不建议作为端侧 Qwen 比赛主线：

| 方案 | 可行性 | 复杂度 | 当前建议 |
| --- | --- | --- | --- |
| Expo Go + JS | 可跑 UI/DeviceMotion | 低 | 已通，继续用于传感器调试 |
| iOS 自定义 MNN/MLX/llama.cpp 模块 | 可行 | 高：需 Mac/Xcode 或 EAS dev client、编 iOS runtime、写 ObjC++/Swift RN 桥、处理模型沙盒与内存 | 不作为当前主线 |
| Android MNN/SME2 | 已有 JNI/Kotlin/RN 模块基础 | 中：需 arm64 真机、模型文件、`-PenableMnn` 构建 | 比赛主线 |

结论：iPhone 先用于“真实数据”；端侧模型先押 Android arm64 真机，能最大化复用现有 `CatuneMnn` 和 SME2/MNN 叙事。

- [ ] x86_64 模拟器无模型时，UI 仍可跑通规则状态机与 F7 模拟数据。
