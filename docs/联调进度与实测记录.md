## 3. 模型下载体验（2026-06-16 新增）

- **断点续传**：状态文件 `mnn_models/.download_state.json`
- **后台下载**：下载逻辑在全局 `modelDownloadService`，切 Tab 不中断；顶部 `ModelDownloadBanner` 显示进度
- **退后台**：按 Home 后进程存活时继续；**系统杀进程**后下次打开 App **自动续传**
- **多模型**：Qwen2.5-0.5B / Qwen3-1.7B 可选；`.active` 标记当前推理模型
- **已安装检测**：完整/不完整/使用中；按钮「下载 / 继续 / 更换 / 设为此模型 / 删除」

## 7. Mac Docker SME2 演示（A1/A2 · 2026-06-20）

> 环境：M2 Mac + OrbStack Docker（`ubuntu:24.04` arm64 镜像 `catune-sme2`）  
> 脚本：`scripts/sme2-demo/run-sme2-min.sh`（A1）、`run-sme2-demo.sh`（A2）  
> 模型：`MNN/qwen2.5-0.5b/`（本地 hf-mirror 已下齐）

### 7.1 A1 最小对照（`run-sme2-min.sh`）

| 步骤 | 结果 | 预期 | 结论 |
| --- | --- | --- | --- |
| Docker 构建 | ✅ ~37min apt | — | 镜像可用 |
| GCC `-march=armv9.2-a+sme2` | ❌ 不支持 | 含 SME 指令 | Ubuntu 24.04 GCC 13 无 `+sme2`，退回纯 HWCAP 检测 |
| A 原生（M2 容器） | `SME:no` `SME2:no` | SME2:no | ✅ 符合 |
| B `qemu -cpu max` | `SME:yes` `SME2:no` | SME2:yes + 指令 OK | △ 仅一半；无 `smstart/smstop executed OK` |
| 脚本退出码 | 2 | 0 | B 未检测到 SME2 |

### 7.2 A2 全链路 MNN（`run-sme2-demo.sh`）

| 项 | 结果 |
| --- | --- |
| 编译 | 容器内 `cmake -DMNN_SME2=ON -DMNN_KLEIDIAI=ON` → `llm_demo` 成功 |
| 推理参数 | `MAX_NEW_TOKENS=24`（修复默认 512 token 重复刷「膳食宝塔」） |
| MNN 能力探测 | `The device supports: ... sme2: 0`（qemu 下仍无 SME2） |
| 中文输出 | 「坐起来放松放松，别紧张。」+ 指南引用尾巴（24 token 打满） |
| 性能（qemu） | prefill ~10.4 tok/s，decode ~5.0 tok/s（**仅功能演示，不作加速口径**） |
| grep 证据 | 仅匹配到 `sme2: 0`，**无** KleidiAI/backend 选中日志 |

**A2 可录**：cmake SME2 开关 + 中文推理 + token 统计。  
**A2 不可声称**：QEMU 已走 SME2 内核 / 已获 SME2 加速。

### 7.3 SME2「运行证据」怎么办（行动清单）

MNN 官方验收标准（见 `docs/【指南】在_MNN_框架中开启_SME2_加速_CPU_推理_v1-1 1.txt`）：运行时 log 出现 **`sme2: 1`**，且 Prefill/Decode 有可对比的加速。

| 优先级 | 做法 | 预期产物 | 难度 |
| --- | --- | --- | --- |
| **P0** | 借 **`hw sme2=true`** 的 arm64 真机（vivo/OPPO Arm C1 等已商用 SME2 机型；或赛事方/demo 机） | App 基准：`hw sme2=true`、`backend=SME2`；`adb logcat \| grep -i sme2` → `sme2: 1` | 中（需设备） |
| **P0** | 同上设备跑 **`llm_bench` / `llm_demo`**（`/data/local/tmp`，同 APK 内 libMNN） | 命令行 `The device supports: ... sme2: 1` + TPS | 中 |
| **P1** | 同设备 **Qwen3-1.7B** + 基准 x2，对比小米14（NEON ~63–88 tok/s） | 「NEON vs SME2」TPS 对比表（唯一合法加速叙事） | 中 |
| **P2** | 升级 Docker：GCC 14+ `+sme`、QEMU 9+（或 `-cpu max,sme2=on`）重跑 A1/A2 | 有望 B 段 `SME2:yes`；**仍慢，且不如真机** | 高、收益低 |
| **P3** | Apple **M4+** Mac 原生编 MNN 跑 `llm_demo` | 可能 `sme2:1`（Apple 从 M4 起有 SME）；与 Android 叙事需分开说明 | 中（若有 M4 机器） |
| **❌ 无效** | 小米14 / Mac QEMU / 模拟器 | 已实测 `sme2:0` | — |

**初赛可交的 SME2 材料（无 L3 设备时）**：
1. Settings 截图：`lib sme2=true` + `hw sme2=false` + readiness 文案  
2. CMake / 编译日志：`MNN_SME2=ON`、KleidiAI 对象  
3. Mac A2 终端：中文推理成功（标注 `sme2:0` 与环境限制）  
4. 口播：「已按 Arm×MNN 官方两步开启；运行时由 MNN 自动探测，当前演示机无 SME2 硬件暴露，故走 NEON；SME2 真机就绪即可加速」
