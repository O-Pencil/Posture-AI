## 2. 模型基准测试（2026-06-18 采样）

### 2.1 基准 x2 结果（3 轮：1 warmup + 2 timed）

| 轮次 | 类型 | Decode TPS | Tokens | 总耗时 |
| --- | --- | --- | --- | --- |
| #1 | warmup | 53.90 tok/s | 17 | 520 ms |
| #2 | timed | 71.60 tok/s | 10 | 283 ms |
| #3 | timed | 54.34 tok/s | 47 | 1077 ms |
| **平均** | timed 仅 | **62.97 tok/s** | — | — |

### 2.2 指标解读（修复 UI 前的问题 & 正确口径）

| 现象 | 结论 |
| --- | --- |
| 旧 UI 顶部 TPS=62.97，Tokens=47，总耗时=1077 ms | **口径混用**：TPS 是 #2+#3 平均，Tokens/总耗时来自 #3 末轮 |
| 旧 UI TTFT=0 ms | **精度丢失**：native 曾用 `ttfa_us/1000` 整除，<1 ms 显示为 0 |
| 47÷1077≈43.6 ≠ 54.34 | **正常**：TPS 只统计 decode 阶段，不是 tokens÷总耗时 |
| 输出远超 30 字 | **模型行为**：0.5B 指令遵循弱，非指标 bug |
| 中文可读、无乱码 | **端侧 LLM 功能验收通过**（对比模拟器 INT4 乱码） |

### 2.3 UI 改进（2026-06-18 起）

- 推理指标区分 **「单次推理」** / **「基准末轮 #N timed」**
- 展示 **TTFT / Prefill / Decode / 总耗时 / 末轮 Decode TPS** 分列
- 基准汇总单独展示 **平均 Decode TPS（不含 warmup）**
- 轮次明细增加 `prefill / decode` 拆分
- Native：`ttft_ms` 等改为 **小数毫秒**，避免 TTFT 被截成 0

> **需重新编译 APK**（`-PenableMnn=true`）后才能在真机看到精度修复。

## 3. Desk 主视觉：当前版本与流畅度反馈

### 3.1 当前代码版本（`main` @ 2026-06-18）

| 模式 | 实现 | 默认 | 提交 |
| --- | --- | --- | --- |
| **雪碧图** | `CatSprite` + `lean_atlas.png`（8×8，60 帧，~6MB） | ✅ 默认 | `0ecc497` |
| **关键帧（逐帧）** | `CatFlipbook` + 60 张 `lean_stage_*.png` | 右上角可切换对比 | 更早 |
| 回退 | `portal.png` 静态图 | 无图集/无帧时 | — |

打包命令：`node scripts/pack-atlas.mjs lean`

### 3.2 实测现象：UI「一闪一闪」、不流畅

| 场景 | 现象 | 原因分析 |
| --- | --- | --- |
| **关键帧 / 逐帧模式** | 角度变化时猫图 **闪烁、卡顿** | 每帧切换 `Image` source → **重新解码 PNG + React 重渲染**（已知根因，见 `CatSprite.tsx` 注释） |
| **雪碧图模式（默认）** | 理论上应流畅（UI 线程平移、单次解码） | 若仍闪：可能是 **首次加载 6MB atlas 解码**、或误切到「关键帧」、或 Metro/Expo Go 未打进最新 atlas |
| **Expo Go vs 原生 APK** | Expo Go 需 Metro 拉 bundle；原生 APK atlas 已打包 | 原生 APK 首屏更稳定；Expo Go 依赖网络与缓存 |

### 3.3 建议操作（验证流畅度）

1. Desk 页确认右上角为 **「雪碧图」**（非「关键帧」）
2. 原生 APK 安装含 `lean_atlas.png` 的版本（`0ecc497` 及之后）
3. 首次进入 Desk 等待 1～2 秒 atlas 解码完成后再录 demo
4. 录 demo 时 **不要** 切到关键帧模式（该模式仅作 A/B 对比）

### 3.4 后续优化方向（待做）

| 优先级 | 项 | 说明 |
| --- | --- | --- |
| P1 | 默认隐藏「雪碧图/关键帧」切换 | 避免 demo 误触回退到逐帧 |
| P1 | Atlas 预加载 / 占位 | Desk mount 时 `Image.prefetch`，解码完成前显示静态首帧 |
| P2 | 缩小图集或 WebP | 减首帧解码时间（当前 6MB PNG） |
| P2 | 真机录屏对比 | 雪碧图 vs 关键帧 同角度滑动各 10s |
