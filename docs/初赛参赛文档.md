## 二、创意参赛文档

### 2.1 应用场景
面向**久坐办公 / 学习人群**的「不驼背坐姿助手」。手机或姿态带实时感知用户的颈椎前倾、胸椎驼背、腰椎侧倾，异常时**端侧 AI 教练**即时给出有温度的提醒与纠正动作，并串起「检测 → 提醒 → 训练 → 复盘」的健康闭环，配「坐姿小植物」养成与成就激励长期坚持。全程核心能力**本地离线可用**。

### 2.2 用户痛点
- **驼背/头前倾不自知**：低头看屏数小时，等到酸痛才意识到，已成习惯。
- **冷冰冰的提醒没人听**：传统提醒只会弹「你驼背了」，机械、无温度、易被无视。
- **隐私顾虑**：体态/健康数据上传云端让人不安。
- **断网即失效**：纯云方案在地铁、会议室等弱网场景不可用。
- **千人一面**：不记得用户偏好与高发问题，建议泛泛而谈。

### 2.3 技术方案

#### 模型选型
| 用途 | 模型 | 量化 | 说明 |
| --- | --- | --- | --- |
| 实时教练文案（主） | **Qwen2.5-0.5B-Instruct** | INT4 | 体积小、真机可读中文、低延迟（✅）；并**搭好 LoRA 微调链路**调成坐姿教练（≤30字+具体动作+`[动作:]`标签，🟡 待转 MNN 真机，见状态总表） |
| 文案加速验收 | Qwen3-1.7B | INT4 | 大模型 + SME2 真机验收 |
| 体态视觉评估 | 云 Qwen-VL（默认）/ 端侧 Qwen2-VL-2B（可选） | INT4 | 侧身照→结构化体态观察；**初赛默认联网评估**（稳、不拖慢文本教练），端侧为可选升级、离线降级预置；复赛做"文本/视觉双模型各司其职" |
| 云端兜底/增强 | 云 Qwen-VL（OpenAI/DashScope 兼容，BYO Key） | — | 视觉评估实时、长报告；**仅增强、可降级到预置**，现场不依赖网络 |

> **AI 与规则的边界（防"为 AI 而 AI"）**：姿态分类 / 打分 / 提醒触发用**规则状态机**（确定、0 延迟、可靠底线）；**模型只做规则做不到的事** —— 自然语言教练话术、视觉体态评估、个性化。模型失败一律回退规则，文案过禁词安全链。

#### 推理框架
- **MNN（MNN-LLM）** 端侧 CPU 推理。`libMNN.so` 自编（`-DMNN_BUILD_LLM=true` + KleidiAI/SME2），约 5.1MB。
- **C++ 推理核** `eyes_llm_session` → **JNI 桥** `eyes_mnn_bridge` → **Kotlin** `MnnPerceptionEngine` → **RN 模块** `CatuneMnn`（`getStatus / inferText / inferTextStream / analyzeImage / runBenchmark`）。
- 业务逻辑（状态机/打分/建议/记忆/UI）全在 **TypeScript** 一套，iOS/Android/Web 通用；仅端侧 AI 与真蓝牙下沉原生。

#### 端侧适配思路
- **量化与算子**：INT4 权重；CPU 能力探测（SME2/i8mm/dot/fp16），SME2 在则走 KleidiAI 加速，否则 NEON 回退。
- **内存/稳定**：`use_mmap` + `tmp_path` 降 OOM；下限 token 控制；模拟器仅验 UI/加载，质量以真机为准。
- **流式输出**：JNI 轮询 `partial_` 缓冲 + 事件回调，文案"打字机"式吐字，掩盖端侧生成延迟。
- **设备自适应**：按 RAM/ABI/SME2/存储分级（入门/主流/高性能）推荐合适模型，避免弱机 OOM。
- **推理↔训练对齐**：App 端 `coachPrompt.ts` 与微调数据 `gen_dataset.py` **同格式**（指令+信号行+记忆前缀），保证微调真正迁移。
- **架构纪律**：仪表盘永远走规则**瞬时**显示，绝不等模型；模型只异步替换文案。

### 2.4 创新点
1. **真·端侧大模型实时教练**：Qwen+MNN 在手机 CPU 本地推理，离线可用、~63–88 tok/s，多数方案只调云 API。
2. **为坐姿场景 LoRA 微调 → 转 MNN 端侧部署**：从风格基准造数据→训练→合并→`.mnn`，端侧出"≤30字+带动作标签"的专业话术。
3. **语义记忆个性化（本地）**：问卷+👍👎 反馈写入本地记忆，按相关性注入教练 prompt，教练"越用越懂你"；数据**仅存本机**。
4. **一份模型输出同时驱动文字 + 视觉**：输出尾部 `[动作:颈部回缩]` 标签 → 既显示话术，又高亮猫脊柱节点、一键进对应跟练。
5. **云 + 端混合、可插拔评估**：体态评估后端「端侧VL / 云端 / 预置」一键切，离线零风险。
6. **角度驱动的拟物主视觉**：黑猫雪碧图按传感器角度 UI 线程平移播放（单次解码、零卡顿），点位实时贴合。

### 2.5 预期效果
- **闭环可演示**：检测（实时角度+猫姿态+点位）→ 提醒（异常+建议动作）→ 训练（引导式倒计时跟练）→ 复盘（日报/周报+成长植物）。
- **端侧 AI 可证真**：基准面板实测 TPS/TTFT/backend/SME2 判定；模型本地加载与推理输入输出现场可见。
- **个性化可感知**：问卷/反馈后，教练话术随记忆变化（带模型时）。
- **离线可靠**：断网仍能分类、打分、提醒、跟练、复盘。

### 4.2 技术难点与解决方案
| 难点 | 解决 |
| --- | --- |
| **SME2 硬件稀缺**（演示机 `hw sme2=false`） | libMNN 按 KleidiAI/SME2 适配编译，有 SME2 自动加速、无则 NEON 回退；定位为「代码已适配，受支持设备即生效」 |
| 端侧生成慢、体感延迟 | 仪表盘走规则瞬时显示，模型异步替换；**流式打字**掩盖延迟 |
| 0.5B 指令遵循弱（输出超长/偏题） | **LoRA 微调**强约束格式与语气；**推理 prompt 与训练逐字对齐** |
| 主视觉逐帧换图卡顿 | 改**雪碧图**：单次解码 + UI 线程平移 snap，零撕影 |
| 模拟器乱码 / OOM | INT4 + `use_mmap`/`tmp_path`；模拟器只验 UI/加载，质量以真机 |
| 记忆要个性化又要隐私 | RN 原生轻量语义记忆，**仅本地**、可清空、过禁词链 |
| 云端依赖网络风险 | 评估后端可插拔，云端**可降级预置**，现场不依赖网络；API Key 仅存本机 |

### 4.3 其他补充说明
- **跨平台一套 TS**：业务逻辑/UI 写一次，iOS/Android/Web(RNW) 通用；只有端侧 AI、真蓝牙下沉原生 → 工程可维护性强。
- **隐私优先**：语义记忆、评估 API Key 均仅存设备本地，不上传、不进 git。
- **健康合规**：全程「健康管理辅助，非医疗诊断」；模型输出过禁词链（禁诊断/承诺/营销词）。
- **硬件路线**：当前以手机 IMU 为真实数据源；决赛接 ESP32-S3 + BNO085 三节点姿态带（BOM ~200 元）。
- **可复现**：模型下载、基准测试、雪碧图打包、微调链路均有脚本与文档（`training/README.md`、`scripts/`、`docs/`）。
- **黑猫人格化**：吉祥物是只爱操心的黑猫，教练话术统一带「喵～」口吻——规则兜底文案、端侧模型系统提示词、微调数据三处口径一致，让"AI 教练"有温度、好记。
