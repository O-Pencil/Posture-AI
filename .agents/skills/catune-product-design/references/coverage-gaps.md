# Coverage Gaps

Use this when a design question has no current Catune standard.

## Known Gaps

- No formal accessibility checklist beyond basic labels, touch targets, and readable layout.
- No screenshot regression harness yet.
- No device-specific viewport matrix yet.
- No finalized rule for when Monitor details should appear on Desk.
- No finalized visual spec for destructive confirmations.
- Pressable accessible-name warnings are not yet enforced as failures.

## How To Add A Standard

Add a rule only after it is accepted in work:

- scope
- reason
- where it applies
- where it does not apply
- good example
- bad example
- whether it should become lint/testable
