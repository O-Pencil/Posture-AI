# Posture-AI · Design DNA

> 项目：Posture-AI（智能体 + 端侧姿态动捕）
> 创建时间：2026-06-05
> 工具：oh-my-design `/teach` 工作流输出

---

## 1. 产品定位（Identity）

Posture-AI 是一款**端侧优先、隐私优先**的智能体应用，把"高频 IMU 脊柱动捕 + 端侧多模态大模型"打包成 MCP 工具。它既是**个人健康教练**（实时检测 + 振动反馈），也是**LLM 智能体的身体代理**（被远程 PC 智能体调用）。

**核心用户**
- **主**：久坐人群（程序员、设计师、学生、办公室职员）—— 习惯看屏幕、不爱动、对"姿势分"敏感。
- **次**：康复期用户（术后 / 慢痛）—— 需要可量化的日常数据。
- **第三**：AI 极客 / Agent 玩家 —— 希望通过 MCP 把"身体"接入 Claude Code / Codex。

**用户状态**：长时间伏案 → 状态分心 + 颈椎疲劳 + 不愿意切换 App → 想要"装上就忘了、有事才提醒"的体验。

## 2. 行业画像（Industry Profile）

**行业**：Consumer Health + Wearable + AI Agent

**核心参考（不抄，只对齐气质）**
- **Oura Ring**：克制、数据即美、不打扰。
- **Fitbit Premium 报告**：把数字翻译成"今天你做了什么"的故事。

- **Linear / Raycast**：Agent/工具向产品的现代感（不学花哨，只学密度）。

**反例（不学）**
- ❌ 医疗类 App 的冰冷白底 + 灰色图标
- ❌ 健身类 App 的荧光黄 + 肌肉图
- ❌ "AI 味"重的渐变紫 + 玻璃拟物 + 居中卡片

## 3. 人格双轴（Personality Axes）

| 轴 | 位置 | 含义 |
| --- | --- | --- |
| **Bold ↔ Restrained** | **Restrained 偏中** | 信息密度优先，不靠堆效果撑场面。状态色克制（健康时只显示数字，不跳弹窗） |
| **Warm ↔ Professional** | **Warm 偏中** | 不冷冰冰。配色带"温感"（暖中性 + 琥珀强调色），但保持医疗级的可读性 |

**一句话调性**：「克制的医疗设备，带一点人体工学椅的暖意」。

## 4. 配色系统（Color System）

### 4.1 状态色（Status · 强约定）

| 名称 | Hex | 用途 |
| --- | --- | --- |
| `status-healthy` | `#10B981`（emerald-500） | Posture Score 90+、NORMAL 状态、推理成功 |
| `status-warning` | `#F59E0B`（amber-500） | TECH_NECK 警告、模型加载中、battery 30% |
| `status-alert` | `#EF4444`（red-500） | SLUMPED/LEFT_LEAN 异常、API 失败、急停 |
| `status-offline` | `#6B7280`（gray-500） | 传感器断开、MCP 关闭、OFFLINE 状态 |
| `status-active` | `#FBBF24`（amber-400） | 当前选中 / 高亮 Mock Console 按钮 |

> **强约束**：状态色不混用。NORMAL 永远是绿，TECH_NECK 永远是琥珀，SLUMPED 永远是红。**不许用渐变**。

### 4.2 表面色（Surface · 暗色优先）

App 端主推**深色**（夜间伏案不刺眼 + OLED 省电），浅色作为可选模式。

| Token | Hex | 用途 |
| --- | --- | --- |
| `bg-canvas` | `#0A0F1A` | 整页底色（带极淡冷调） |
| `bg-surface-1` | `#111827` | 卡片底色 |
| `bg-surface-2` | `#1F2937` | 浮起层 / 二级卡片 |
| `bg-surface-raised` | `#0F172A` | 顶栏 / 粘性头 |
| `border-subtle` | `#1E293B` | 1px 卡片描边 |
| `text-primary` | `#F1F5F9` | 主文字 |
| `text-secondary` | `#94A3B8` | 次文字（标签、单位） |
| `text-tertiary` | `#64748B` | 三级（辅助说明） |
| `accent-warm` | `#F97316`（orange-500） | 品牌强调（CTA、Logo 点缀） |

> **反 AI Slop**：绝不用 `#000` 纯黑或 `#fff` 纯白；所有背景都带极淡的色相（这里偏冷蓝）。

### 4.3 数据可视化色板（Charts）

折线图/柱状图按"健康度梯度"排序：
1. `#10B981`（emerald）
2. `#34D399`（emerald-light）
3. `#FCD34D`（amber-light）
4. `#F59E0B`（amber）
5. `#EF4444`（red）

## 5. 字体（Typography）

| 用途 | 字体 | 来源 | 备选 |
| --- | --- | --- | --- |
| UI 主字体 | **Geist Sans** | Vercel 开源，可 Google Fonts | system-ui（无网兜底） |
| 显示 / 数字 | **Geist Mono** | Vercel | JetBrains Mono |
| 中文回退 | **PingFang SC** → **Noto Sans SC** | 系统 / Google | system-ui |
| 不使用 | Inter / Roboto / Arial（AI Slop 警告） | — | — |

**字号阶梯**（rem 基准 16px）：
- `text-display` 60px · Posture Score 主数字
- `text-h1` 32px · 页面标题
- `text-h2` 24px · 卡片标题
- `text-h3` 18px · 区块小标题
- `text-body` 15px · 正文
- `text-meta` 13px · 标签、单位
- `text-caption` 11px · 角注

**字重**：400（正文）、500（小标题）、700（强调）、800（Posture Score 大数字）。

---

## 6. 间距与栅格（Spacing & Grid）

**4px 基础栅格**，常用档位：`4 · 8 · 12 · 16 · 20 · 24 · 32 · 48 · 64 · 96`。

**移动端断点**（按内容驱动，不按设备）：
- `≥ 320px`：可用
- `≥ 380px`：标准手机（iPhone 14/15 标准宽度）
- `≥ 480px`：大手机 / 小平板
- `≥ 768px`：横屏 / 折叠屏

**圆角**：`rounded-xl`（12px）主用，`rounded-2xl`（16px）大卡片，`rounded-full`（胶囊）按钮与状态徽章。**不使用 0px 直角**。

**阴影**：极克制，仅 modal 用 `shadow-2xl`，卡片用 `ring-1 ring-white/5` 替代阴影。

## 7. 组件模式（Component Patterns）

| 组件 | 模式 |
| --- | --- |
| 按钮（主） | `bg-accent-warm text-white rounded-full px-6 py-3 font-medium` |
| 按钮（次） | `bg-surface-2 text-primary rounded-full px-5 py-2.5` |
| 按钮（危险） | `bg-status-alert/10 text-status-alert border border-status-alert/30` |
| 状态徽章 | `inline-flex items-center gap-1.5 rounded-full px-2.5 py-0.5 text-meta font-medium` + 圆点 |
| 卡片 | `bg-surface-1 ring-1 ring-border-subtle rounded-2xl p-5` |
| 进度环 | SVG circle + `stroke="url(#grad)"` 24% → 4px 描边 |
| 列表项 | `flex items-center justify-between py-3 border-b border-border-subtle/50` |
| 输入框 | `bg-surface-2 border-b border-border-subtle focus:border-accent-warm` |
| Tab | `inline-flex bg-surface-2 rounded-full p-1` + 选中态 `bg-surface-raised` |
| 模态 | `bg-surface-1 ring-1 ring-border-subtle rounded-3xl p-6 shadow-2xl` |

## 8. 动效（Motion）

| 场景 | 时长 | 缓动 |
| --- | --- | --- |
| 状态切换（颜色/数字） | 200ms | `ease-out` |
| 卡片进入 | 300ms | `ease-out`，`translateY(8px) → 0` |
| 模态弹出 | 250ms | `ease-out-quart`，`scale(0.95) → 1` + fade |
| 数字滚动 | 600ms | `ease-in-out`，分段 |
| 振动反馈 | 由原生 API 控制 | 不用 UI 动效代替 |

> **禁用** bounce、elastic、bouncy-bezier。**禁用** > 600ms 的过渡。**禁用** `transition: all`（必须指定属性）。

---

## 9. 反模式（Forbidden）

照搬自 oh-my-design 通用反模式 + 本项目额外约束：

- ❌ 紫色→蓝色渐变（AI Slop 标志）
- ❌ Inter / Roboto / Arial 为主字体
- ❌ 玻璃拟物 backdrop-filter（除非 modal）
- ❌ 居中对称"3 张卡"布局
- ❌ 灰色文字放在彩色背景上
- ❌ `#000` / `#fff` 纯色背景
- ❌ 粒子 / 流光装饰
- ❌ "10K+ 用户" / "99.9% 正常" hero
- ❌ `background-clip: text` 渐变文字
- ❌ 卡片里再套卡片
- ❌ **本项目额外**：状态色用渐变（永远实色）
- ❌ **本项目额外**：把 Posture Score 做成大渐变数字（永远实色 + Geist Mono）
- ❌ **本项目额外**：超过 2 个弹窗同时出现

---

## 10. 可访问性（A11y）

- 对比度：正文 ≥ 4.5:1，大字 ≥ 3:1（已通过配色预设达成）
- 触控目标 ≥ 44×44 pt
- 所有 icon button 必须有 `aria-label`
- `prefers-reduced-motion`：禁用所有非必要动画
- 数字必须带单位（`°`、`%`、`ms`）
